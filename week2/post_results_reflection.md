# Week 2 Post-Results Reflection

Week 2 confirmed that the tighter trust-region strategy was the right adjustment for this stage of the challenge. The main change from Week 1 was moving away from broader surrogate-driven suggestions and toward a more selective policy: exploit tightly where the previous round improved performance, refine cautiously where the result was plausible but not better, and return to the best historical basin where the latest move had underperformed.

The most useful operational addition this week was the sanity-check step. Before locking the final submission, each proposed query was checked for distance from the best historical point, whether it stayed inside the intended trust region, how strong the nearest observed outputs were, and whether any feature had been pushed to a boundary without enough evidence. This made the process more disciplined and helped separate candidates that were merely imperfect from candidates that were too aggressive for a limited remaining query budget.

That review directly affected Function 5. The original candidate was in a strong region, but the checks showed it had drifted outside the intended trust region and pushed one coordinate to the upper boundary. The final submission replaced it with a safer local exploitation point that preserved the same high-performing pattern while staying closer to the validated basin. The returned output for Function 5 was the strongest result of the week, which supports that change.

Across the functions, the outcomes reinforced the decision to use different tactics by group. Functions 2, 5, 7, and 8 responded well, which suggests their best-known basins are now credible enough to justify continued tight local exploitation. Functions 1 and 3 improved only modestly, so they still look more like cautious refinement cases. Functions 4 and 6 underperformed, which suggests the next round should reset toward the best historical region rather than continue in the same direction.

Looking ahead to Week 3, the plan is to keep the same evidence-driven workflow while narrowing it further. The strongest functions should be treated as momentum cases and refined in very small neighbourhoods. The weaker functions should either stay close to a known good region or explicitly reset to the best historical point. At this stage, the emphasis is less on exploring the whole landscape and more on using each remaining query as carefully as possible.
